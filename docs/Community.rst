SCONE community
===============

The Goal of this document is to state the principles that should guide the
SCONE community.

When the SCONE was started, the primary motivation was 'fun'.
Monte Carlo codes are objectively cool! It is fun to write them.
It is fun to experiment with the methods.

SCONE exists to spread this joy to as many people as possible!

We should be welcoming to all people of all backgrounds.
We should strive to help each other. We need to be open to any new idea and 
anyone offering their effort and time to help the code.

The direction of the SCONE development should be decided by consensus.
If agreement cannot be reached we should strive for a compromise.

Reviewers and Maintainers
-------------------------

Every person with a major contribution to the code should be given a 'Reviewer'
status. Their approval is required for any PR to be merged. They should be 
given write access to the repository. And allowed to merge PRs.

The merging and management of the repository is the responsibility of
the 'maintainers'. Any 'reviewer' after sensible period of activity should 
be given 'maintainer' level access if they wish.

The role of the 'Maintainer' is to keep the repository in a healthy state and 
manage the correct permissions.
